================================================================================
                            1. 서버팩 소개
================================================================================

이 서버팩은 마인크래프트 1.21.8에서 작동하는 하드코어 생존 서버를 위해 제작되었습니다.

【제작 목적】
- 일반 생존 서버에 긴장감과 협동 요소 추가
- 사망에 의미를 부여하고 동료 구출 시스템 제공
- 한글 닉네임을 자유롭게 사용할 수 있는 시스템 제공
- 플레이어 간 협력을 유도하는 부활 메커니즘 구현

【주요 기능】
✓ 사망 시 관전 모드 전환 및 시체 생성
✓ 부활 토큰 및 생명의 메아리를 통한 부활 시스템
✓ 시체 회수 및 재설치 시스템
✓ 한글 닉네임 시스템 (최대 11개 슬롯 저장)
✓ 닉네임 머리 위 표시 (벽 너머에서도 보임)
✓ 회차 관리 및 월드 리셋 기능

================================================================================
                            2. 설치 방법
================================================================================

【서버 설치】
1. 이 폴더의 모든 파일을 서버 디렉토리에 복사합니다
2. eula.txt 파일을 생성하고 "eula=true"를 입력합니다
3. start.bat 파일을 생성하고 다음 내용을 입력합니다:

   @echo off
   java -Xmx4G -Xms2G -jar server.jar nogui
   pause

4. start.bat을 실행하여 서버를 시작합니다

【필요 사항】
- Java 21 이상
- Paper 1.21.8 서버 엔진 (포함되지 않음)
- Skript 플러그인 2.12.2 (포함됨)
- skript-reflect 플러그인 2.6.1 (포함됨)
- 최소 4GB RAM 권장

【서버 첫 실행】
서버가 처음 실행되면 다음 메시지가 표시됩니다:
- "하드코어 생존기 시스템 활성화!"
- "하드코어 생존기 제작 시스템 활성화!"
- "하드코어 생존기 명령어 시스템 활성화!"
- "하드코어 생존기 관리자 시스템 활성화!"

- "한글 닉네임 시스템 활성화!"

================================================================================
                         3. 게임 시스템 설명
================================================================================

【하드코어 생존 시스템】

1) 사망 시스템
   - 플레이어가 사망하면 즉시 관전 모드로 전환됩니다
   - 사망 위치에 시체(아머스탠드)와 통(아이템 저장)이 생성됩니다
   - 사망한 플레이어는 다른 플레이어에게 보이지 않습니다
   - 위험한 죽음(용암, 낙사, 익사 등)은 스폰으로 부활합니다

2) 부활 시스템 (2가지 방법)

   A. 부활 토큰 사용
      - /revive 명령어로 자가 부활 가능
      - 토큰 1개 소모
      - 일반 죽음: 시체 위치로 부활
      - 위험한 죽음: 스폰으로 부활
      - 부활 시 체력/배고픔 50%, 저항/구속/화염저항 효과 부여

   B. 생명의 메아리 사용
      - 다른 플레이어가 아이템을 우클릭하여 부활
      - 10블록 내의 시체를 찾아 부활시킴
      - 부활 시 저항 III, 구속 II (5초) 부여
      - 일회용 아이템 (사용 후 소멸)

3) 시체 회수 시스템
   - 나무 막대기로 시체 우클릭 시 머리 아이템 획득
   - 5분 쿨타임 적용
   - 머리를 다시 설치하면 시체가 재생성됩니다
   - /gethead 명령어로도 회수 가능

4) 통 저장 시스템
   - 사망 시 아이템이 통에 자동 저장
   - 부활 후 시체 위치에서 아이템 회수 가능
   - 관리자는 /hcbarrel로 ON/OFF 가능

【한글 닉네임 시스템】

1) 닉네임 변경
   - /nmchange <닉네임> 으로 닉네임 변경
   - 탭 리스트, 채팅, 사망 메시지에 모두 적용
   - 머리 위에 텍스트 디스플레이로 표시
   - 벽 너머에서도 닉네임 보임

2) 닉네임 저장 슬롯
   - 0~10번 총 11개의 슬롯에 닉네임 저장 가능
   - /nmsave <번호> <닉네임> 으로 저장
   - /nmload <번호> 로 불러오기
   - /nmremove <번호> 로 제거
   - /nmlist <플레이어> 로 목록 확인

3) 닉네임 초기화
   - /nmreset 으로 원래 닉네임으로 복구

================================================================================
                      4. 명령어 가이드
================================================================================

┌─────────────────────────────────────────────────────────────────────────┐
│                   4.1 일반 플레이어 명령어 (하드코어)                    │
└─────────────────────────────────────────────────────────────────────────┘

/hcstatus
  - 현재 하드코어 생존 상태 확인
  - 회차, 생존/사망 상태, 부활 토큰 개수 표시

/revive
  - 부활 토큰 1개를 소모하여 자가 부활
  - 2번 확인 필요 (10초 이내 재입력)
  - 일반 죽음: 시체 위치로 부활
  - 위험한 죽음: 스폰으로 부활

/heads
  - 모든 플레이어의 상태 목록 확인
  - 생존/사망 상태, 토큰 개수 표시
  - 머리 회수 쿨타임 표시

/gethead <플레이어>
  - 사망한 플레이어의 머리 회수
  - 5분 쿨타임 적용
  - 시체(아머스탠드) 자동 제거
  - 예시: /gethead Steve

/craftecho [개수/max]
  - 생명의 메아리 제작
  - 개수 생략 시 1개 제작
  - max 입력 시 최대 개수 제작
  - 2개 이상 제작 시 2번 확인 필요
  - 예시: /craftecho 5, /craftecho max

/hchelp
  - 하드코어 생존 명령어 전체 도움말 표시

┌─────────────────────────────────────────────────────────────────────────┐
│                   4.2 일반 플레이어 명령어 (닉네임)                      │
└─────────────────────────────────────────────────────────────────────────┘

/nmchange <닉네임>
  - 닉네임 변경
  - 예시: /nmchange 홍길동

/nmsave <번호> <닉네임>
  - 닉네임을 슬롯에 저장 (0~10)
  - 예시: /nmsave 1 홍길동

/nmload <번호>
  - 저장된 닉네임 불러오기
  - 예시: /nmload 1

/nmlist <플레이어>
  - 플레이어의 닉네임 목록 확인
  - 예시: /nmlist Steve

/nmremove <번호>
  - 저장된 닉네임 제거
  - 예시: /nmremove 1

/nmcheck <닉네임>
  - 닉네임의 원래 플레이어 확인
  - 예시: /nmcheck 홍길동

/nmreset
  - 닉네임 초기화 (원래 닉네임으로)

/nmhelp
  - 닉네임 시스템 전체 도움말 표시

┌─────────────────────────────────────────────────────────────────────────┐
│                    4.3 관리자 명령어 (하드코어)                          │
│                    권한: hardcore.admin                                  │
└─────────────────────────────────────────────────────────────────────────┘

/setround <회차>
  - 현재 회차 설정
  - 예시: /setround 1

/givetoken add/set <플레이어> <개수>
  - 특정 플레이어에게 토큰 지급
  - add: 현재 토큰에 추가
  - set: 토큰 개수를 특정 값으로 설정
  - 예시: /givetoken add Steve 5
  - 예시: /givetoken set Steve 10

/givetokenall add/set <개수>
  - 모든 플레이어에게 토큰 지급
  - 예시: /givetokenall add 5
  - 예시: /givetokenall set 10

/makeecho [개수]
  - 생명의 메아리 직접 지급 (최대 128개)
  - 개수 생략 시 1개
  - 예시: /makeecho 10

/testgethead <플레이어>
  - 강제 머리 획득 (쿨타임 무시)
  - 관리자 전용 테스트 명령어
  - 예시: /testgethead Steve

/hcbarrel
  - 통 저장 시스템 ON/OFF 토글
  - ON: 사망 시 아이템이 통에 저장
  - OFF: 사망 시 아이템이 바닥에 드롭

/hcdebug
  - 디버그 모드 ON/OFF 토글
  - 디버그 메시지 표시 여부 설정

/hcreset
  - 월드 완전 리셋 (되돌릴 수 없음!)
  - 2번 확인 필요 (10초 이내 재입력)
  - 모든 시체, 토큰, 인벤토리, 데이터 삭제
  - 회차 자동 증가
  - 수동으로 월드 폴더 삭제 필요

/cleancorpses
  - 모든 시체(아머스탠드) 제거
  - 시체 정리용

/clearheads <플레이어>
  - 특정 플레이어의 머리 아이템 전체 제거
  - 모든 플레이어 인벤토리에서 검색 및 제거
  - 예시: /clearheads Steve

/checkduplicates
  - 중복된 머리 아이템 수동 검사
  - 디버그용 명령어

┌─────────────────────────────────────────────────────────────────────────┐
│                    4.4 관리자 명령어 (닉네임)                            │
│                    권한: nickname.admin                                  │
└─────────────────────────────────────────────────────────────────────────┘

/nmadminforce <플레이어> <닉네임>
  - 강제로 플레이어의 닉네임 변경
  - 예시: /nmadminforce Steve 홍길동

/nmadminclear <플레이어> <번호>
  - 플레이어의 저장된 닉네임 슬롯 제거
  - 예시: /nmadminclear Steve 1

/nmadminreset <플레이어>
  - 플레이어의 현재 닉네임 초기화
  - 예시: /nmadminreset Steve

/nmadminclearslots <플레이어>
  - 플레이어의 저장된 닉네임 전체 제거 (0~10번 슬롯)
  - 예시: /nmadminclearslots Steve

================================================================================
                          5. 제작 시스템
================================================================================

【생명의 메아리 제작법】

필요 재료 (1개당):
  - 구리 주괴 x1
  - 철 주괴 x1
  - 금 주괴 x1
  - 다이아몬드 x1
  - 바다의 심장 x1
  - 레드스톤 x1
  - 청금석 x1
  - 에메랄드 x1
  - 네더라이트 파편 x1

제작 방법:
  1. 위 재료를 인벤토리에 준비합니다
  2. /craftecho [개수] 명령어를 입력합니다
  3. 2개 이상 제작 시 10초 이내에 재확인 필요
  4. 제작이 완료되면 Echo Shard 아이템을 받습니다

사용 방법:
  1. 생명의 메아리를 들고 우클릭
  2. 10블록 내의 시체를 자동으로 찾아 부활시킵니다
  3. 부활한 플레이어는 저항 III, 구속 II (5초) 효과를 받습니다
  4. 아이템은 사용 후 소멸됩니다

   - 잘못된 명령어 감지 및 제안

3. hardcore-admin.sk
   - 관리자 명령어
   - 토큰 지급, 회차 설정, 월드 리셋

4. hardcore-crafting.sk
   - 생명의 메아리 제작 시스템

5. hardcore-duplicate-check.sk

   - 한글 닉네임 시스템

   - 채팅, 탭 리스트, 사망 메시지 적용

================================================================================
                           7. 문제 해결
================================================================================

【Q1. 서버가 시작되지 않습니다】
A: Java 21 이상이 설치되어 있는지 확인하세요.
   java -version 명령어로 버전을 확인할 수 있습니다.

【Q2. 스크립트가 로드되지 않습니다】
A: plugins 폴더에 Skript-2.12.2.jar와 skript-reflect-2.6.1.jar가
   있는지 확인하세요. 서버를 재시작해보세요.

【Q3. 닉네임이 머리 위에 표시되지 않습니다】
A: 서버를 재시작하거나 /nmreset 후 다시 닉네임을 설정해보세요.

【Q4. 시체가 너무 많이 쌓였습니다】
A: 관리자 권한으로 /cleancorpses 명령어를 사용하세요.

【Q5. 부활 토큰이 작동하지 않습니다】
A: /hcstatus로 토큰 개수를 확인하세요.
   사망 상태인지 확인하세요 (is_dead 태그).

【Q6. 머리 회수 쿨타임을 확인하고 싶습니다】
A: /heads 명령어로 모든 플레이어의 쿨타임을 확인할 수 있습니다.

【Q7. 월드를 리셋하고 싶습니다】
A: 1) /hcreset 명령어를 2번 입력합니다 (10초 이내)
   2) /stop으로 서버를 종료합니다
   3) world, world_nether, world_the_end 폴더를 삭제합니다
   4) 서버를 재시작합니다

【Q8. 권한을 설정하고 싶습니다】
A: LuckPerms 등의 권한 플러그인을 사용하여 다음 권한을 설정하세요:
   - hardcore.admin : 하드코어 관리자 권한
   - nickname.admin : 닉네임 관리자 권한
